# Training Dashboard Roadmap V1

## Purpose

This archived roadmap records the first product sequence that moved the app from a training log toward a lightweight coaching workflow.

## Status

This roadmap generation is complete.

## Covered Phases

## Phase 1: Make Adaptive Planning Usable In-App

Goal: remove the dependency on chat for common replanning actions.

Summary:

- in-app remaining-week adjustment
- clearer comparison semantics
- plan revision visibility

## Phase 2: Add The Feedback Loop

Goal: make future recommendations depend on how workouts actually felt, not just what was logged.

Summary:

- post-workout feedback
- recovery-aware planning signals

## Phase 3: Link Goals To Planning

Goal: make plans reflect goals instead of existing as a separate layer.

Summary:

- goal-aware weekly plans
- goal progress forecast

## Phase 4: Improve Data Quality And Structure

Goal: reduce ambiguity in how planned and completed sessions relate.

Summary:

- planned-to-actual linking
- structured workout intent

## Phase 5: Make Coaching Easier Through MCP

Goal: reduce prompt overhead and make chat interactions more reliable.

Summary:

- one-shot coaching MCP action
- plan-diff confirmation flow
- roadmap visibility
- coaching history and revision timeline
- multi-week execution analysis
- stronger deterministic coaching heuristics
- goal progress and planning forecasts
- modality restrictions and injury-aware coaching
